Data Types and Data Structures

Python Variables¶

  • are assigned with =

  • are dynamically typed (have no static type)

  • are pointers

  • variable names:

    • must begin with a letter (lower or upper case) or underscore _

    • may contain letters, numbers and underscore, but cannot begin with a number

    • are case sensitive

Python Variables are assigned with =¶

# assign 4 to the variable x
x = 4 
print(x)
4

Python Variables are passed-by-reference¶

In contrast, Java variables are passed-by-value

x = 3.1415   # define a container in memory with value 3.1415 and assign variable/pointer x to it
y = x        # assign a second variable y that points to the same location as x
print(x)
print(y)
print(id(x)) # address of the container x is pointing at
print(id(y))
3.1415
3.1415
1517492552688

x = 2.71     # define a new container with value 2.71 and reassign pointer x to the new location
print(x)
print(y)
print(id(x))
print(id(y))
2.71
3.1415
1517493075120
1517492552688

Note: Java would have modified the same container x changing the value 3.1415 to 2.71.

In contrast, Python creates 2 containers (one with value 3.1415, one with value 2.71) and just reassigns the pointer x.

# Bonus:
x += 3       # this also creates another container with the value 2.71 + 3 = 5.71
print(x)
print(id(x))
5.71
1517493075216

Note: Python differentiates between immutable objects (int, float, bool, str, tuple) and mutable ones (list, dict, set).

More on this later.

Dynamic Typing¶

  • Type only known at runtime

  • Can result in semantic errors

# a function for multiplying numbers by two:
def multiply_by_two(x):
    return x * 2

x = 2 # x is an integer
print(multiply_by_two(x))
4

x = "2" # x is a string
print(multiply_by_two(x))
22

  • ‘Normal’ (that is static) typing: variable is declared to be of certain type

int c = 0;

  • Python does not have variables with static types (but see mypy)

String Formatting¶

'This is a bad approximation of pi: 3.14159 but better than 3'

'Both of these approximations of pi are bad: 3.142 and 3'

Scenario 1¶

Scenario 1¶
# What's happening here?
a = [1, 2, 3, 5] # Define a list container and point at it through pointer a
print(a)
b = a       # A new pointer b pointing to the same container as pointer a
print(b)    # Pointer b points to the same container as a
a = a + [8] # From right to left: access container from a, extend it with '8', store somewhere else in memory 
            # and reassign pointer a to the new location 
print(a)    # Pointer a points to the new extended container
print(b)    # Pointer b points to the original container
[1, 2, 3, 5]
[1, 2, 3, 5]
[1, 2, 3, 5, 8]
[1, 2, 3, 5]

Note: Adding / concatenating lists together via + creates a new list.

Sets¶

  • Unordered collections of unique items

  • Support set operations

  • Defined by {1, 2, 3}

primes = {2, 3, 5, 7}
odds = {1, 3, 5, 7, 9}

Union¶

items appearing in either set

primes = {2, 3, 5, 7}
odds = {1, 3, 5, 7, 9}
primes | odds      # with an operator
primes.union(odds) # equivalently with a method
{1, 2, 3, 5, 7, 9}

Intersection¶

items appearing in both sets

primes = {2, 3, 5, 7}
odds = {1, 3, 5, 7, 9}
primes & odds             # with an operator
primes.intersection(odds) # equivalently with a method
{3, 5, 7}

Difference¶

items appearing in one but not other set

primes = {2, 3, 5, 7}
odds = {1, 3, 5, 7, 9}
primes - odds           # with an operator
primes.difference(odds) # equivalently with a method
{2}

Symmetric difference¶

items appearing in only one set

primes = {2, 3, 5, 7}
odds = {1, 3, 5, 7, 9}
primes ^ odds                     # with an operator
primes.symmetric_difference(odds) # equivalently with a method
{1, 2, 9}

  • Hash table

  • Fast access

  • Defined by key:value pairs within curly braces: {'a':1, 'b':2, 'c':3}

Anaconda setup¶

Change path to new conda env¶

  • Click on Configure Conda

  • Add or change the envs_dirs as needed

  • Any new conda envs created in the future will be in this directory.

Create new conda env with Python 3.9¶

  • Go to Environments

  • Click on Create

  • Select Python 3.9 and then Create

  • If you get errors, don’t panic. Try this command in the conda shell: conda clean --all

  • After moving to the new env with a different Python version, back in the Anaconda Navigator Home, you may see Install instead of Launch for Jupyter Notebook, CMD.exe Prompt, etc. That is OK. Simply install again and you will be able to launch them.

Optional but very useful: change base dir for Jupyter Notebook¶

  • Generate the jupyter_notebook_config.py with this command in the shell:

jupyter notebook --generate-config

  • Uncomment the setting c.ServerApp.root_dir and specify a folder you prefer

  • Make sure any backslashes are escaped!
